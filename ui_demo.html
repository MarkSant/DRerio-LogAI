Implementation Details

  • Analysis Interval: Process every N frames (detection, recording)
  • Display Interval: Update GUI every M processed frames (independent of analysis)
  • Defaults: Both intervals default to 10 frames
  • Persistence: Settings saved in project data and per-video project_settings.json
  • Single Video: Intervals passed through config to processing pipeline
  • Project Batch: Intervals read from project data

Processing Flow Changes

Before: Process every frame, display coupled to processing

After: Process every analysis_interval_frames, display every display_interval_frames processed frames
