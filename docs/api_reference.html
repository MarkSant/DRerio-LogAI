API Stability Contract (v4.0)

Status: Stable
Last Updated: 23/11/2025
Module: src/zebtrack/ui/gui.py

This document defines the public API of the ApplicationGUI class. With the v4.0 refactoring (Event-Driven Architecture), the public API has been drastically reduced, delegating responsibilities to specialized internal components. Only essential methods for initialization, lifecycle, and global feedback remain.

1. Lifecycle & Initialization

__init__(root, controller, event_bus=None, settings_obj=None)

Initializes the application, builds UI components, and sets up the UICoordinator.

2. Feedback Delegates

These methods delegate to DialogManager, keeping a convenient facade for global messages.

show_error(title, message)
show_warning(title, message)
show_info(title, message)
ask_ok_cancel(title, message)
ask_string(title, prompt, initialvalue=None)
ask_directory(title)
ask_open_filenames(title, filetypes)
ask_save_filename(**options)

3. Status & Progress

Methods for updating the global status bar and progress indicators.

set_status(text)
show_progress_bar()
hide_progress_bar()
update_progress(value)
update_idletasks()

4. UI State Control

update_button_state(button_name, state)

Controls the enabled/disabled state of global buttons (Record, Process, Stop).

5. Specific Interfaces (Wizards)

ask_recording_details_unified()
ask_missing_metadata(experiment_id)
show_pending_videos_dialog(...)

6. Controller Integration (Callbacks)

These methods are called by the Controller to update the visualization in real-time.

update_weights_dropdown(weights)
set_active_weight_in_dropdown(weight_name)
update_openvino_checkbox(enabled)
update_openvino_status_display(status)
update_gpu_hardware_display(hardware_summary)
update_detection_overlay(detections, report)
update_processing_mode(report)
update_analysis_profile(profile_name)
update_analysis_progress(value, status_text)
update_analysis_metadata(metadata)
setup_zone_definition_for_single_video(video_path, config)

⚠️ Removed/Migrated Methods (v3 → v4)

The following methods are NO LONGER part of the public GUI API. They have been moved to specialized components or replaced by events.

  • update_zone_listbox → UIEvents.ZONES_UPDATED
  • refresh_project_views → UIEvents.PROJECT_VIEWS_REFRESH_REQUESTED
  • apply_pending_readiness_snapshot → UIEvents.READINESS_SNAPSHOT_UPDATED
  • _edit_selected_zone_vertices → CanvasManager.edit_selected_zone_vertices
  • _on_canvas_click → CanvasEventHandler
  • _format_day_display → ValidationManager
  • _pipeline_summary_exists → ProjectViewManager
  • _generate_unified_report → ProjectViewManager

Policy on Changes

  1. No New Public Methods: Avoid adding public methods to GUI. Prefer adding logic to Managers (CanvasManager, etc.) or UICoordinator.
  2. Event-Driven: Communication between components must occur EXCLUSIVELY via EventBusV2.
  3. Container Pattern: The GUI should act only as a container and dependency injection point.
