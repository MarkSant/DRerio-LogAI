# Migration Guide: v2.1 → v3.0

## Overview

ZebTrack-AI v3.0 removes deprecated APIs and introduces breaking changes. This guide helps you migrate smoothly.

**Timeline:**

- **v2.1** (November 2025): Deprecation warnings added
- **v3.0** (February 2026): Deprecated features removed

---

## Breaking Changes

### 1. Reporter Constructor (CRITICAL)

**Removed:** Direct instantiation with `trajectory_df`

**Before (v2.1):**

```python
from zebtrack.analysis.reporter import Reporter

reporter = Reporter(
    trajectory_df=df,
    metadata={"experiment_id": "exp_001"},
    pixelcm_x=10.0,
    pixelcm_y=10.0,
    video_height_px=480,
    freezing_threshold=1.0,
    # ... many more parameters ...
)
```

**After (v3.0):**

```python
from zebtrack.analysis.analysis_service import AnalysisService
from zebtrack.analysis.reporter import Reporter
from zebtrack.settings import load_settings

# Step 1: Run analysis via service
settings = load_settings()
service = AnalysisService(settings_obj=settings)

analysis_result = service.run_full_analysis_as_dto(
    trajectory_df=df,
    pixelcm_x=10.0,
    pixelcm_y=10.0,
    video_height_px=480,
    arena_polygon_px=arena,
    rois=rois,
    fps=30.0,
    freezing_vel_threshold=1.0,
    freezing_min_duration=0.5
)

# Step 2: Create reporter from result
reporter = Reporter.from_analysis(analysis_result)
```

**Benefits:**

- ✅ **Performance:** Analysis computed once, cached in DTO
- ✅ **Type Safety:** DTO validates all parameters
- ✅ **Testability:** Easier to mock AnalysisResult
- ✅ **Consistency:** Same analysis used for reports and exports

**Automated Migration:**

```bash
# Run migration script
poetry run python scripts/migrate_reporter_v3.py --check  # Dry run
poetry run python scripts/migrate_reporter_v3.py --apply  # Apply changes
```

### 2. Settings Singleton Removal

**Removed:** Global `settings` object

**Before (v2.1):**

```python
from zebtrack import settings  # ❌ No longer exists

camera_index = settings.camera.index
```

**After (v3.0):**

```python
# In main entry point (__main__.py):
from zebtrack.settings import load_settings

settings_obj = load_settings()

# Inject into services:
camera = Camera(settings_obj=settings_obj)
detector_service = DetectorService(..., settings_obj=settings_obj)
```

**Migration Steps:**

1. Find all `from zebtrack import settings` imports
2. Replace with `from zebtrack.settings import Settings` (for type hints only)
3. Accept `settings_obj` parameter in constructor
4. Update composition root to inject settings

**Example - Service Migration:**

**Before:**

```python
from zebtrack import settings

class MyService:
    def __init__(self):
        self.confidence = settings.yolo_model.confidence_threshold
```

**After:**

```python
from zebtrack.settings import Settings

class MyService:
    def __init__(self, settings_obj: Settings):
        self.confidence = settings_obj.yolo_model.confidence_threshold
```

**Example - Composition Root:**

```python
# In __main__.py
from zebtrack.settings import load_settings

# Load settings once
settings_obj = load_settings()

# Inject into all services
state_manager = StateManager()
detector_service = DetectorService(settings_obj=settings_obj)
video_service = VideoProcessingService(settings_obj=settings_obj)
project_manager = ProjectManager(settings_obj=settings_obj)

# Create MainViewModel with injected dependencies
main_view_model = MainViewModel(
    state_manager=state_manager,
    detector_service=detector_service,
    video_service=video_service,
    project_manager=project_manager,
    settings_obj=settings_obj,
    # ... other dependencies
)
```

## Test Migration

### Update Fixtures

**Before:**

```python
@pytest.fixture
def reporter():
    return Reporter(trajectory_df=sample_df, ...)
```

**After:**

```python
@pytest.fixture
def analysis_result():
    # Create AnalysisResult DTO
    from zebtrack.analysis.models import AnalysisResult, CalibrationParams
    from zebtrack.analysis.behavior import ConcreteBehavioralAnalyzer
    
    calibration_params = CalibrationParams(
        pixelcm_x=10.0,
        pixelcm_y=10.0,
        video_height_px=480,
        arena_polygon_px=[(0, 0), (640, 0), (640, 480), (0, 480)],
        fps=30.0,
    )
    
    return AnalysisResult(
        report={"summary": {}},
        behavioral_analyzer=None,  # Or mock
        roi_analyzer=None,
        trajectory_df=sample_df,
        metadata={"experiment_id": "test"},
        calibration_params=calibration_params,
        rois=[],
        roi_colors={},
    )

@pytest.fixture
def reporter(analysis_result):
    return Reporter.from_analysis(analysis_result)
```

### Settings Fixtures

**Before:**

```python
def test_my_service():
    service = MyService()  # Uses global settings
    assert service.confidence > 0
```

**After:**

```python
@pytest.fixture
def mock_settings():
    from zebtrack.settings import Settings, YOLOModelSettings
    
    return Settings(
        yolo_model=YOLOModelSettings(confidence_threshold=0.5),
        # ... other required fields
    )

def test_my_service(mock_settings):
    service = MyService(settings_obj=mock_settings)
    assert service.confidence == 0.5
```

## Common Patterns

### Pattern 1: Service Initialization

**Before:**

```python
# Service reads from global singleton
class VideoProcessingService:
    def __init__(self):
        from zebtrack import settings
        self.fps = settings.video.target_fps
```

**After:**

```python
# Service receives settings via constructor
class VideoProcessingService:
    def __init__(self, settings_obj: Settings):
        self.fps = settings_obj.video.target_fps
```

### Pattern 2: Accessing Settings in Methods

**Before:**

```python
def process_frame(self, frame):
    from zebtrack import settings
    threshold = settings.processing.threshold
    # ... use threshold
```

**After:**

```python
# Store settings reference in __init__
def __init__(self, settings_obj: Settings):
    self.settings = settings_obj

def process_frame(self, frame):
    threshold = self.settings.processing.threshold
    # ... use threshold
```

### Pattern 3: Optional Settings

**Before:**

```python
def analyze(self, use_gpu=None):
    from zebtrack import settings
    if use_gpu is None:
        use_gpu = settings.hardware.use_gpu
```

**After:**

```python
def __init__(self, settings_obj: Settings):
    self.settings = settings_obj

def analyze(self, use_gpu: bool | None = None):
    if use_gpu is None:
        use_gpu = self.settings.hardware.use_gpu
```

## FAQ

### Q: Why was the singleton removed?

**A:** Singletons create hidden dependencies that make code harder to test and reason about. Dependency injection makes dependencies explicit and enables better unit testing with mocks.

### Q: Can I still use direct Reporter instantiation in v2.1?

**A:** Yes, but you'll see deprecation warnings. We recommend migrating to `from_analysis()` now to prepare for v3.0.

### Q: Do I need to update my config.yaml?

**A:** No, the configuration file format remains the same. Only the Python API changed.

### Q: What if I have custom plugins?

**A:** Custom detector plugins should accept `settings_obj` in their constructor and pass it to the base class. See `docs/detector_service_api.md` for details.

### Q: Performance impact?

**A:** The new pattern is **faster** because analysis is computed once and cached in the DTO, avoiding redundant calculations.
