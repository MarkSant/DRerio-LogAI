# Sprint Reports Index

This directory now groups every sprint-related document so the root of the
repository stays focused on active guides. The content is organized into
clear buckets so you can jump straight to the level of detail you need.

| Folder | Purpose | Typical Files |
| --- | --- | --- |
| `analysis/` | Deep-dive investigations, retro notes, and forensic write-ups for a single sprint. | `SPRINT_8_CLEANUP_ANALYSIS.md`, `sprint26_recording_methods_analysis.md` |
| `plans/` | Playbooks, checklists, and recommendation decks that describe what we planned to execute. | `SPRINT_15_DELEGATION_COMPLETION_PLAN.md`, `SPRINT_27_RECOMMENDATION.md` |
| `overview/` | Concise summaries that merge multiple documents into one executive-friendly brief. | `SPRINTS_15-19_SUMMARY.md`, `SPRINT_27_SUMMARY.md` |
| `SPRINT_XX_RESULTS.md` | Final sprint results that already live under `docs/sprints/` | `SPRINT_27_RESULTS.md`, `SPRINT_33_RESULTS.md` |

## How to use this folder

1. **Start with an overview** (for example `overview/SPRINT_27_SUMMARY.md`) to
   understand the outcome and current status at a glance.
2. **Drill into `analysis/`** if you need the raw investigation details or the
   metrics that back the recommendations.
3. **Review `plans/`** whenever you need the actionable checklist, extraction
   order, or mitigation strategy that was agreed upon during the sprint.

If you add new sprint material, please place it in the right subfolder and
reference it from the overview to keep the structure tidy.
