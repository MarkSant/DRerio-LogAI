# Contributing

Contributions are welcome!

We appreciate all contributions, from bug reports to new features. To get started, please feel free to open an issue to discuss any changes you would like to make.

## Development Process

1. Fork the repository.
2. Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name` or `git checkout -b fix/your-bug-fix`.
3. Make your changes and commit them with a clear message.
4. Push your changes to your fork.
5. Open a pull request to the main repository.

We will review your pull request as soon as possible. Thank you for your interest in improving this project!
