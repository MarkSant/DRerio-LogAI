Plano de Projeto Detalhado: ZebTrack AI
1. Sumário Executivo
Visão do Projeto: Consolidar os projetos PyZebArdYolo e ZebAdapt numa única aplicação Python de alto desempenho, denominada ZebTrack AI. Este software unificado irá abranger todo o fluxo de trabalho da pesquisa comportamental, desde a aquisição de vídeo e controlo de hardware até à análise de dados complexa e visualização interativa, eliminando a dependência do MATLAB e introduzindo capacidades avançadas de IA.
Objetivos Estratégicos:
1. Unificação Tecnológica: Migrar 100% da lógica de análise do ZebAdapt (MATLAB) para um novo módulo em Python, integrado de forma nativa no PyZebArdYolo.
2. Aprimoramento Científico: Substituir a deteção por frame por um algoritmo de rastreamento multi-animal robusto, que mantenha a identidade individual ao longo do tempo.
3. Otimização de Desempenho: Implementar um formato de dados moderno (Apache Parquet) para acelerar drasticamente as operações de leitura e análise de dados.
4. Melhoria da Usabilidade (UX): Desenvolver uma interface gráfica dedicada à análise e visualização de dados, tornando a exploração dos resultados mais intuitiva e interativa.
5. Inovação: Integrar agentes de IA generativa para automatizar a configuração de protocolos e a geração de relatórios interpretativos.
6. Acessibilidade: Empacotar a aplicação final como um executável autónomo, permitindo a sua fácil distribuição e utilização por colaboradores sem conhecimentos de programação.

   * 1.2. Definição do Padrão de Dados Unificado
   * Ação 1.2.1: Formalizar a estrutura de dados. Criar um documento em docs/data_schema.md que descreva cada coluna do ficheiro Parquet, o seu tipo de dados e o seu significado.
   * frame_id: int64 - Índice numérico sequencial do frame de vídeo.
   * timestamp: float64 - Tempo em segundos desde o início da gravação, para cálculos cinemáticos precisos.
   * animal_id: int32 - Identificador único e persistente para cada animal, atribuído pelo rastreador.
   * x_px, y_px: float32 - Coordenadas do centroide do bounding box em píxeis.
   * bbox_w_px, bbox_h_px: float32 - Largura e altura do bounding box em píxeis.
   * confidence_score: float32 - Pontuação de confiança (0.0 a 1.0) da deteção original.
   * roi_name: string (object) - Nome da Região de Interesse (ROI) onde o centroide do animal está localizado. None se estiver fora de todas as ROIs.
   * Justificativa Técnica: A formalização do esquema previne ambiguidades e serve como um "contrato" entre o módulo de gravação e o módulo de análise. A escolha de tipos de dados específicos (ex: float32 em vez de float64 para coordenadas) otimiza o espaço em disco sem perda de precisão necessária.
   * 1.3. Implementação do Rastreamento Multi-Objeto Persistente
   * Local: src/zebtrack/plugins/yolo_detector.py.
   * Ação 1.3.1: No config.yaml, adicionar uma secção para o rastreador: tracker: botsort.yaml. Isto permite flexibilidade para testar outros rastreadores (ex: bytetrack.yaml) no futuro.

   * Ação 1.3.3: O objeto results retornado por .track() tem uma estrutura diferente. O loop de pós-processamento deve ser reescrito para aceder aos IDs de rastreamento, que estão em results[0].boxes.id. Se um ID não estiver presente para uma deteção, essa deteção deve ser descartada.
   * Critério de Sucesso: Ao processar um vídeo com dois ou mais animais que se cruzam, o ficheiro .parquet de saída mostra que os animal_ids são mantidos corretamente antes e depois da oclusão.
   * 1.4. Otimização do Módulo de Gravação de Dados
   * Local: src/zebtrack/io/recorder.py.
   * Ação 1.4.1: Na classe DataRecorder, no método __init__, inicializar um self.buffer = [] e um self.buffer_size_limit = 100.
   * Ação 1.4.2: No método que recebe os dados de cada frame (ex: record_data), em vez de escrever diretamente no disco, a lógica será: self.buffer.append(data_dict).
   * Ação 1.4.3: Adicionar uma verificação: if len(self.buffer) >= self.buffer_size_limit: self.flush_buffer_to_disk().
   * Ação 1.4.4: Criar o método flush_buffer_to_disk(). Este método converterá self.buffer num DataFrame do Pandas, escreverá para o ficheiro Parquet e depois limpará o buffer (self.buffer = []).
   * Ação 1.4.5: Garantir que o método close() da classe DataRecorder chame self.flush_buffer_to_disk() uma última vez para gravar quaisquer dados remanescentes.
   * Critério de Sucesso: Durante uma gravação, o monitor de atividade do disco mostra escritas intermitentes (em rajadas) em vez de uma escrita contínua, e a aplicação mantém uma taxa de frames por segundo (FPS) estável.
Fase 2: Desenvolvimento do Módulo de Análise Comportamental
Objetivo: Migrar a lógica científica do ZebAdapt para um módulo Python robusto, validado e extensível.
   * 2.1. Criação da Arquitetura de Análise
   * Ação 2.1.1: Criar o ficheiro src/zebtrack/analysis/analyzer.py.
   * Ação 2.1.2: Implementar a classe BehavioralAnalyzer. O seu construtor (__init__) receberá os caminhos para os ficheiros de dados e de configuração, e carregará imediatamente os dados para um DataFrame do Pandas (self.df).
   * Ação 2.1.3: Criar um método run_analysis_pipeline() que chamará sequencialmente cada método de análise individual (ex: _calculate_kinematics, _calculate_zone_occupancy). Os resultados de cada análise serão armazenados num dicionário self.results.
   * 2.2. Migração e Validação das Métricas de Análise
   * Ação 2.2.1 (Análise Espacial):
   * No método _calculate_zone_occupancy, carregar as coordenadas das ROIs do config.yaml.
   * Para cada ROI, criar um objeto shapely.geometry.Polygon.
   * Usar self.df.apply(lambda row: self._get_roi_for_point(row['x_px'], row['y_px']), axis=1) para preencher a coluna roi_name. O método _get_roi_for_point iterará sobre os polígonos e retornará o nome do primeiro que contiver o ponto.
   * Cálculo de Métricas: Utilizar pandas.groupby(['animal_id', 'roi_name']) para calcular o tempo total (.size() * (1/FPS)), e pandas.crosstab para a matriz de transição entre zonas.
   * Referência MATLAB: areasequence_computation.m, desenha_areas.m.
   * Ação 2.2.2 (Análise Cinemática):
   * No método _calculate_kinematics, primeiro, aplicar um filtro de suavização. self.df['x_smooth'] = self.df.groupby('animal_id')['x_px'].transform(lambda x: savgol_filter(x, window_length=5, polyorder=2)). Repetir para y_px.
   * Calcular as diferenças (.diff()) nas coordenadas suavizadas e no timestamp para obter dx, dy e dt.
   * A velocidade será np.sqrt(dx**2 + dy**2) / dt. Armazenar numa nova coluna velocity_px_s.
   * Referência MATLAB: associateeuclid.m.
   * Critério de Sucesso: Criar um teste unitário em tests/test_analyzer.py com um pequeno ficheiro Parquet de entrada e um resultado esperado conhecido. O teste deve comparar a saída do BehavioralAnalyzer com os valores esperados, com uma tolerância de 0.01.
